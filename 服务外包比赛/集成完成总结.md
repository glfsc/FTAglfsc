## 🎯 核心功能实现

### 1. 文件上传功能

**API**: `POST /api/v1/fault-tree/upload`

- ✅ 支持多种文件格式（PDF、TXT、DOCX、MD 等）
- ✅ 自动生成唯一文件名防止冲突
- ✅ 保存到 `uploads/` 目录
- ✅ 返回 file_id 供后续使用

### 2. 知识提取功能

**API**: `POST /api/v1/fault-tree/extract`

- ✅ 使用 Qwen LLM 提取故障三元组
- ✅ 自动验证 JSON 格式和数据完整性
- ✅ 批量插入到 Neo4j 知识图谱
- ✅ 转换为前端兼容的事件/逻辑门格式
- ✅ 输出 JSON 文件到 `data/output/`（用于测试）
- ✅ 自动清理缓存确保数据新鲜度

### 3. 故障树生成功能

**API**: `GET /api/v1/fault-tree/generate_tree?top_event=xxx`

- ✅ 从知识图谱提取因果路径
- ✅ 自动破环和逻辑门修复
- ✅ 计算树状布局坐标
- ✅ 生成前端可视化 JSON
- ✅ 支持导出为文件

## 🔄 完整工作流程

```
用户操作
   ↓
[1. 上传文件] → POST /api/v1/fault-tree/upload
   ↓ 返回 file_id
[2. 提取知识] → POST /api/v1/fault-tree/extract
   ↓ 自动执行:
     - 读取文件内容
     - LLM 提取三元组
     - 验证和解析
     - 插入知识图谱
     - 转换为事件/逻辑门
     - 清理缓存
   ↓ 返回 triplets, events, gates
[3. 生成故障树] → GET /api/v1/fault-tree/generate_tree?top_event=xxx
   ↓ 自动执行:
     - 从图谱查询路径
     - 构建 NetworkX 图
     - 计算布局坐标
     - 生成 JSON
   ↓ 返回可视化数据
[4. 前端渲染] → 使用 D3/AntV 渲染故障树
```

## 🔍 故障排查

### 问题 1: LLM 调用失败

**症状**: `Exception: API 调用失败`

**解决**:
1. 检查 `DASHSCOPE_API_KEY` 是否正确
2. 检查网络连接
3. 查看模型配额是否充足

### 问题 2: 文件读取失败

**症状**: `RuntimeError: 文件读取失败`

**解决**:
1. 检查文件格式是否支持
2. PDF 文件需要安装 pdfplumber
3. DOCX 文件需要安装 python-docx

### 问题 3: Neo4j 连接失败

**症状**: `ConnectionError: 数据库连接失败`

**解决**:
1. 确认 Neo4j 服务正在运行
2. 检查 `.env` 中的连接信息
3. 确认防火墙未阻止 7687 端口
