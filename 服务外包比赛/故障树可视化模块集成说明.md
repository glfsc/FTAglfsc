# 故障树可视化模块集成说明

## 概述

已成功将故障树可视化模块整合到主项目中，提供以下功能：

### 核心功能
1. **交互式故障树编辑** - 支持添加、删除、修改节点
2. **AI 智能分析** - 基于 Qwen-max 模型的对话式分析
3. **多种导出格式** - PDF、PNG、JSON、Word 文档
4. **自动布局** - 基于 Dagre 算法的树状布局
5. **逻辑门显示** - AND/OR 逻辑门可视化

## 使用方法

### 方式一：从生成页面跳转

1. 访问 `/upload` 上传文件并提取知识
2. 访问 `/generate` 生成故障树
3. 点击 **"可视化故障树"** 按钮进入可视化页面
4. 系统会自动加载最近生成的故障树

### 方式二：直接访问

访问 `/visualization` 直接打开可视化编辑器

### 主要功能操作

#### 1. 查看故障树
- 鼠标拖拽：移动画布
- 滚轮缩放：放大/缩小视图
- 点击节点：选中并查看详情

#### 2. 编辑节点
- **添加节点**：点击左上角 "Add Event" 按钮
- **修改节点**：点击节点后在右侧面板编辑
- **删除节点**：选中节点后点击 "Delete" 按钮
- **连接节点**：从源节点的底部拖拽到目标节点的顶部

#### 3. AI 对话
- 点击右上角 "AI Assistant" 按钮
- 输入问题，例如：
  - "分析顶事件的概率"
  - "创建发动机故障的故障树"
  - "添加一个基本事件：传感器失效"
- AI 会自动分析并提供建议，甚至可以直接生成新的故障树

#### 4. 导出数据
- 点击 "Export" 按钮选择导出格式：
  - **PDF**：打印版结构图
  - **PNG**：高清图片
  - **JSON**：原始数据
  - **Word**：分析报告

## API 接口说明

### AI 对话接口

**端点：** `POST /api/v1/ai/chat`

**请求体：**
```json
{
  "messages": [
    {
      "role": "user",
      "content": "分析这个故障树"
    }
  ],
  "context": {
    "nodes": [...],
    "edges": [...]
  }
}
```

**响应：**
```json
{
  "response": "AI 的回答内容",
  "success": true
}
```

### 特性说明

1. **上下文感知**：AI 会读取当前故障树结构进行分析
2. **JSON 生成**：当用户要求修改/创建故障树时，AI 会返回 JSON 格式的完整结构
3. **自动更新**：前端检测到 JSON 块后会自动解析并更新画布

## 常见问题

### Q1: 为什么看不到可视化按钮？
A: 确保已经在 `/generate` 页面成功生成了故障树，并且数据已保存到 sessionStorage。

### Q2: AI 对话不工作？
A: 检查以下几点：
- 确认 `.env` 中的 `DASHSCOPE_API_KEY` 是否正确
- 检查后端服务是否运行
- 查看浏览器控制台是否有错误信息

### Q3: 节点重叠怎么办？
A: 点击画布空白处重新触发 `fitView()`，或手动拖拽调整位置。

### Q4: 如何导入已有的故障树？
A: 点击 "Import" 按钮，选择之前导出的 JSON 文件即可。
