## 🚀 快速开始

### 1️⃣ 安装测试依赖

```bash
cd backend
pip install -r requirements-test.txt
```

需要的依赖：
- pytest: 测试框架
- pytest-cov: 覆盖率报告
- pytest-asyncio: 异步测试支持
- httpx: HTTP 客户端（用于 API 测试）

### 2️⃣ 运行测试的三种方式

#### 方式一：使用批处理脚本（推荐）

```bash
cd backend
.\运行测试.bat
```

这会显示一个交互式菜单，可以选择：
- ✅ 运行所有测试
- ✅ 按模块运行测试
- ✅ 生成覆盖率报告
- ✅ 清理测试缓存

#### 方式二：使用 pytest 命令

```bash
# 运行所有测试
pytest

# 运行特定模块
pytest test/api_tests/
pytest test/service_tests/
pytest test/model_tests/

# 运行单个测试文件
pytest test/api_tests/test_ai_chat.py -v

# 运行单个测试方法
pytest test/api_tests/test_ai_chat.py::TestAIChatAPI::test_basic_chat -v
```

#### 方式三：在 IDE 中运行

如果使用 VSCode 或 PyCharm：
- VSCode: 安装 Python 扩展后，测试文件会显示运行按钮
- PyCharm: 右键点击测试文件选择 "Run" 或 "Debug"

## 📝 测试模块说明

### API 测试 (`api_tests/`)

测试 REST API接口的功能

**示例：test_ai_chat.py**
```python
def test_basic_chat(self):
    """测试基础聊天功能"""
    request_data = {
        "messages": [{"role": "user", "content": "你好"}]
    }
    response = self.client.post("/api/v1/ai/chat", json=request_data)
    assert response.status_code == 200
```

**包含的测试：**
- ✅ 基础聊天功能

## 🏷️ 使用标记筛选测试


```python

    pass
```

## 💡 最佳实践

### 1. 测试命名规范

```python
# 文件名
test_<module>.py

# 类名
Test<Module>

# 方法名
test_<feature>_<scenario>()

# 示例
test_ai_chat.py
class TestAIChatAPI:
    def test_basic_chat_with_valid_request(self):
        pass
```

### 2. 使用夹具 (Fixtures)

```python
import pytest

@pytest.fixture
def sample_triplet():
    return create_mock_triplet()

def test_using_fixture(sample_triplet):
    assert sample_triplet["confidence"] == 0.95
```

### 3. 参数化测试

```python
@pytest.mark.parametrize("input,expected", [
    (1, 2),
    (2, 4),
    (3, 6),
])
def test_double(input, expected):
    assert input * 2 == expected
```

### 4. Mock 外部依赖

```python
from unittest.mock import patch

@patch('module.external_api_call')
def test_with_mock(mock_api):
    mock_api.return_value = {"result": "success"}
    # 测试逻辑
```

## ⚠️ 注意事项

1. **环境变量**: 确保 `.env` 文件配置正确
2. **数据库连接**: 部分测试需要 Neo4j 数据库运行
3. **API Key**: AI 相关测试需要 `DASHSCOPE_API_KEY`
4. **测试隔离**: 每个测试应该独立，不依赖其他测试
5. **资源清理**: 使用 `teardown` 或 `yield fixture` 清理资源

## 📈 持续集成

### GitHub Actions 配置

```yaml
name: Tests

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      
      - name: Set up Python
        uses: actions/setup-python@v2
        with:
          python-version: 3.8
      
      - name: Install dependencies
        run: |
          pip install -r requirements.txt
          pip install -r requirements-test.txt
      
      - name: Run tests
        run: pytest --cov=app --cov-report=xml
```

## ❓ 常见问题

### Q: ImportError 错误？
A: 确保在项目根目录运行 pytest，或调用 `add_project_to_path()`

### Q: 如何跳过某些测试？
A: 使用 `@pytest.mark.skip` 装饰器

```python
@pytest.mark.skip(reason="暂未实现")
def test_future_feature():
    pass
```

### Q: 测试异步代码？
A: 使用 `pytest-asyncio`

```python
import pytest

@pytest.mark.asyncio
async def test_async_function():
    result = await async_func()
    assert result == "expected"
```
