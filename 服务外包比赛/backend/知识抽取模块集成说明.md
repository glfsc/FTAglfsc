# 知识抽取模块集成说明

## 概述

已成功将知识抽取模块（triplet_extractor）集成到主项目中，实现了从文档中自动提取故障知识三元组的功能。

## 主要改动

### 1. 新增服务层

**文件**: `backend/app/services/knowledge_extraction_service.py`

- 封装了 QwenExtractor 的调用逻辑
- 支持多种文件格式（TXT、PDF、DOCX、MD 等）
- 提供三元组的提取、验证和保存功能

### 2. 更新 FaultTreeService

**文件**: `backend/app/services/fault_tree_service.py`

主要变更：
- 注入 KnowledgeExtractionService
- 重写 `extract_knowledge()` 方法，整合真实的 LLM 提取逻辑
- 添加三元组到事件/逻辑门的转换方法（兼容旧版前端）
- 自动清理缓存，确保生成时获取最新数据

### 3. 新增 API 接口

**文件**: `backend/app/api/v1/routes/fault_tree.py`

新增接口：

#### 3.1 上传文件
```http
POST /api/v1/fault-tree/upload
Content-Type: multipart/form-data

参数：file (文件)

响应：
{
  "file_id": "abc123...",
  "filename": "test.pdf",
  "file_size": 1024,
  "status": "success",
  "message": "文件上传成功"
}
```

#### 3.2 提取知识
```http
POST /api/v1/fault-tree/extract
Content-Type: application/json

请求体：
{
  "file_id": "abc123...",
  "mode": "multimodal"  // 可选：legacy 或 multimodal
}

响应：
{
  "task_id": "extract_abc123",
  "triplets": [
    {
      "subject_name": "电源模块故障",
      "subject_type": "BasicEvent",
      "relation": "resultsIn",
      "object_name": "系统宕机",
      "object_type": "TopEvent",
      "confidence": 0.95,
      "source": "设备手册"
    }
  ],
  "events": [...],  // 转换为前端格式的事件
  "gates": [...],   // 转换为前端格式的逻辑门
  "status": "completed",
  "progress": 1.0,
  "output_file": "data/output/test_triplets.json",  // 测试用输出文件
  "accuracy_metrics": {
    "avg_confidence": 0.92
  }
}
```

### 4. 数据模型更新

**文件**: `backend/app/models/schemas.py`

- 更新 `KnowledgeExtractRequest`：支持 mode 参数
- 更新 `KnowledgeExtractResponse`：增加 triplets、output_file 字段
- 保持与旧版前端的事件/逻辑门格式兼容

### 5. 前端适配

**文件**: `frontend/src/api/index.js`

更新 `extractKnowledge` API 调用，支持传入对象参数：
```javascript
export const extractKnowledge = (data) => {
  return api.post('/fault-tree/extract', data)
}
```

**文件**: `frontend/src/pages/ExtractPage.vue`

更新调用方式：
```javascript
const result = await extractKnowledge({
  file_id: fileId.value,
  mode: 'multimodal'
})
```

## 使用流程

### 完整流程（前端）

1. **上传文件** → `/api/v1/fault-tree/upload`
   - 获得 file_id
   
2. **提取知识** → `/api/v1/fault-tree/extract`
   - 传入 file_id
   - 获得三元组、事件、逻辑门
   - 自动写入 Neo4j 知识图谱
   
3. **生成故障树** → `/api/v1/fault-tree/generate_tree?top_event=xxx`
   - 从知识图谱提取数据
   - 生成可视化 JSON

### 测试流程（手动）

1. **上传文件并提取**
   ```bash
   # 上传文件
   curl -X POST http://localhost:8000/api/v1/fault-tree/upload \
     -F "file=@test_document.pdf"
   
   # 提取知识
   curl -X POST http://localhost:8000/api/v1/fault-tree/extract \
     -H "Content-Type: application/json" \
     -d '{
       "file_id": "abc123...",
       "mode": "multimodal"
     }'
   ```

2. **查看输出的 JSON 文件**
   - 位置：`backend/data/output/<文件名>_triplets.json`
   - 包含完整的三元组数据

3. **手动生成故障树**
   ```bash
   curl "http://localhost:8000/api/v1/fault-tree/generate_tree?top_event=系统宕机"
   ```

## 核心代码逻辑

### 知识抽取流程

```python
# 1. 读取文件内容（支持多种格式）
text = self._read_file_content(file_path)

# 2. 调用 Qwen LLM 提取三元组
json_result = self.extractor.extract(text, source_reference=filename)

# 3. 解析和验证 JSON 结果
validated_data = TripletExtractionResult(**parsed_data)

# 4. 批量插入到 Neo4j 知识图谱
inserted_count, skipped, duplicates = kg_builder.insert_triplets_batch(triplets)

# 5. 转换为前端格式（兼容旧版）
events = _triplets_to_events(triplets)
gates = _triplets_to_gates(triplets)

# 6. 清理缓存，确保下次生成时获取最新数据
tree_generator.clear_cache(None)
```

### 文件格式支持

- ✅ TXT - 直接读取
- ✅ MD - 直接读取
- ✅ JSON - 直接读取
- ✅ CSV - 直接读取
- ✅ PDF - 使用 pdfplumber 提取文本
- ✅ DOCX - 使用 python-docx 提取文本和表格

## 测试示例

### 示例 1：基本使用

```python
from app.services.knowledge_extraction_service import KnowledgeExtractionService

# 创建服务实例
```

## 错误处理

### 常见错误及解决方案

1. **DASHSCOPE_API_KEY 未设置**
   - 错误：`ValueError: DASHSCOPE_API_KEY 环境变量未设置`
   - 解决：在 `.env` 文件中配置 API Key

2. **文件格式不支持**
   - 错误：`RuntimeError: 文件读取失败`
   - 解决：检查文件扩展名，确保是支持的格式

3. **Neo4j 连接失败**
   - 错误：`ConnectionError: 数据库连接失败`
   - 解决：检查 Neo4j 服务是否运行，环境变量是否正确

4. **LLM 返回 JSON 格式错误**
   - 错误：`json.JSONDecodeError` 或 `ValidationError`
   - 解决：查看日志中的原始响应，调整 prompt 或联系模型提供商

## 性能优化建议

1. **批量处理**：使用 `insert_triplets_batch()` 而非逐条插入
2. **缓存管理**：知识写入后自动清理缓存，避免脏读
3. **异步处理**：对于大文件，考虑使用 Celery 等异步任务队列
4. **文件存储**：生产环境建议使用对象存储（如 MinIO、S3）
